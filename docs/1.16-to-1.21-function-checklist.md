# 1.16.5 / 1.21.11 功能對照驗收清單

此文件用於 Step 26 最終驗收。請用同一份清單分別在 Paper `1.16.5` 舊版與 Paper `1.21.11` 新版執行，記錄實際行為；不要用記憶或程式碼推測取代實測結果。

## 填寫規則

每個項目都用下列分類之一：

- `一致`：兩版操作結果、訊息、權限、遊戲狀態大致一致。
- `可接受差異`：兩版不同，但不影響玩法，且委託人接受。
- `版本限制造成的差異`：因 Minecraft / Paper 版本差異無法完全一致。
- `升級造成 regression`：新版比舊版壞，需修正或取得明確接受。
- `原版既有問題`：舊版本來就有問題，新版沒有新增問題。

## 測試帳號與權限狀態

只需要兩個實際帳號即可。測試時直接用 `C6Yelan` 和 `_SnYe`，並用 LuckPerms 或既有權限設定切換當下權限狀態，再記錄結果。

建議固定：

- `C6Yelan`：主要操作者，輪流切換成無權限、預設權限、`wonderland.uhc.moderator`、`wonderland.uhc.host`、`wonderland.uhc.admin`。
- `_SnYe`：目標玩家 / 隊友 / 對手，用於 invite、join gate、kill、respawn、viewheal、backpack、switchteam 等需要第二人的項目。
