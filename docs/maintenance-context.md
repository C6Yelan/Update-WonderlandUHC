# WonderlandUHC 維護脈絡摘要

整理日期：2026-05-21

這份文件說明本 repository 的維護脈絡，主要給委託人、原作者與後續維護者理解此更新版本的來源與協作方式。這不是開服安裝教學，也不是 release 使用者的主要入口。

## 專案定位

本 repository 是基於原作者 WonderlandUHC 的 Paper `1.21.11` 更新維護版本。

本次更新工作是在委託脈絡下進行，目標是讓原插件能在新版 Paper / Java 環境中繼續運作，並整理後續維護所需的架構、文件與驗證流程。

這個 fork 不應被描述為原作者本人直接發布的新版本；對外說明時應保留原作者與原專案脈絡，並明確標示這是更新維護版本。

## Codex 協助說明

本次升級過程中，部分程式分析、修改建議、文件整理、測試指令執行與驗證紀錄使用 OpenAI Codex 輔助完成。

Codex 在本專案中作為開發輔助工具使用，主要協助：

- 閱讀與整理既有程式碼。
- 比對 commit history、文件與實際行為。
- 產生或修改小範圍程式碼與文件。
- 執行封裝、測試與伺服器啟動檢查。
- 協助整理交接文件與摘要文件。

這不代表所有變更皆為自動產生，也不代表可以用工具輸出取代人工判斷。實際維護與發布責任仍應由本 fork 的維護者承擔。

## 判讀變更的依據

若需理解實際變更內容，應優先參考：

1. commit history。
2. 測試與封裝結果。
3. Paper `1.21.11` 實機啟動與回測紀錄。
4. `docs/` 根目錄中的摘要文件。
5. 原始碼與設定檔目前的實際內容。

若摘要文件與實際程式碼不一致，應以目前程式碼、測試結果與最新 commit 為準，並回頭修正文檔。

## 維護原則

後續維護時建議維持以下原則：

1. 尊重原作者與原專案脈絡。
2. 對 fork 的更新內容、限制與相容範圍保持明確說明。
3. 不把 Codex 或其他工具視為變更責任主體。
4. 重要修改需保留可追溯紀錄，例如 commit、測試結果或文件摘要。
5. 若對外發布 release，README 與 release note 應以開服者實際需要的資訊為主。

修改後需要哪些封裝、啟動與流程驗證，見 `docs/details/verification.md`。
